为了避免上面这种情况的发生，postman 诞生了。Postman 是一款在开发人员中非常流行的 API 调试工具，简单、方便。下面来简单学习下使用 postman 测试接口吧

postman 的简单使用

先贴上 postman 官网：https://www.getpostman.com/

  • 第一步，下载 PostMan，点击首页的 Download the App，选择需要的版本下载

  • 下载好后，打开软件，简单的注册，选择个人喜好

  • 进入主界面,点击左上角New按钮，会看到弹框功能提示，

    • Request：模拟各种 HTTP Request，例如常用的 GET、POST、再到 Restful API 的 PUT 等。
    • Collection（测试集合）：是 request 的请求集合，当你在做完一个测试后，可以把这次的 request 存到特定的 collection 里，这样，下次再做同样的测试时，就不需要再次重新输入了。

  • 这里我们选择Request,填写介绍信息

  • 保存之后就会出现如下的界面

  • 在对应位置填写信息

    • 请求方式：POST，GET，PUT。。。
    • 请求地址：请求的 URL 地址。需要注意 postman 中不存在跨域问题，实际请求数据可能要解决跨域
    • 请求参数：params(查询参数),authorization(认证),headers(请求头),body(请求体)

  • 填写完请求信息就可以点击右上角Send按钮发送请求

  • 后台返回的数据会显示在下方Response区域，可以选择pretty方便查看
